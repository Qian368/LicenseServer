# 许可证验证工具接口文档

## 1. 概述

**程序名称**：LicenseServer

**程序类型**：.NET Windows Forms 桌面/控制台应用程序

**核心功能**：许可证验证与管理工具，支持本地验证服务和远程部署API验证两种验证模式。

**主要能力**：

- 硬件绑定：基于CPU、主板生成本机唯一机器码。
- 许可证激活：将许可证与机器码绑定，并管理激活设备数量。
- 后端管理：内置Pocketbase后端管理
- 本地验证：启动内置的PocketBase服务，在本地进行许可证验证。
- 远程验证：连接到指定的远程API服务进行许可证验证，开放自定义验证URL。
- 授权证书：支持服务端验证结果并生成带签名和加密的证书，开放本地证书验证周期端口。

**后端数据库配置**(详见pocketbase\pb\_data\data.db)：

| 表名         | 字段1  | 字段2          | 字段3            | 字段4                        | 字段5             | 字段6                         |
| ---------- | ---- | ------------ | -------------- | :------------------------- | --------------- | --------------------------- |
| `licenses` | `id` | `key`        | `is_active`    | `expires_at`               | `max_devices`   | `devices`                   |
| 推荐类型       | text | text         | bool           | Datetime                   | number          | Relation：`devices:`multiple |
| `devices`  | `id` | `machine_id` | `computer_nam` | `licence`                  | `activate_time` | `updated`（用于排序）             |

## 2. 程序入口与命令行参数

程序主入口为 `MainForm.Main(string[] args)` 方法。支持通过命令行参数控制程序行为，无需打开图形界面。

| `-verifytype` | `/verifytype` | 指定验证类型。`local`：启动本地服务验证并执行验证；`remote`：直接调用远程API验证。 | 如果选择-action为`verify，则必填` | `local` 或 `remote` | local                      | `-verifytype remote`                |

### 2.2 Action 参数详解

`-action` 参数支持以下值：

| Action值     | 功能描述              | 依赖参数                                | 输出说明                                                          |
| ----------- | ----------------- | ----------------------------------- | ------------------------------------------------------------- |
| 交互模式（默认）    | 启动图形界面，支持手动点击按钮操作 | 无                                   | 图形界面展示，操作结果弹窗提示                                               |

## 3. 匿名管道通信接口

### 3.1 接口概述

为实现主程序与许可证验证工具（LicenseServer）的无窗口、低耦合通信，程序提供**匿名管道**通信接口，支持主进程通过管道向LicenseServer发送验证请求，并同步接收验证结果，无需依赖命令行参数，无需处理任何复杂验证逻辑，只需读取结果。

### 3.2 管道模式启用参数

| 参数名         | 缩写/别名       | 说明                                  | 必填 | 值类型      | 默认值 | 示例                      |
| ----------- | ----------- | ----------------------------------- | -- | -------- | --- | ----------------------- |
| `-pipemode` | `/pipemode` | 启用管道通信模式，需跟随两个管道句柄参数（主进程传递的读/写管道句柄） | 是  | 布尔标志+字符串 | 无   | `-pipemode 0x123 0x456` |

#### 参数说明：

- 启用`-pipemode`时，程序会忽略除管道句柄外的其他命令行参数，仅监听管道通信；
- 管道句柄需传递两个：第一个为“主进程写→子进程读”的管道句柄，第二个为“主进程读←子进程写”的管道句柄；
- 管道模式下，程序自动只完成GUI验证对话框及结果交互，不显示验证主窗口，仅处理管道请求。

### 3.3 管道通信数据格式

#### 3.3.1 请求格式（主进程→LicenseServer）

请求为JSON字符串，结构与`PipeCommunication.VerifyRequest`类一致：

```JSON

{
  "VerifyType": "string",  // 必选：验证类型，local/remote
  "ApiUrl": "string",       // 可选：远程验证API地址，仅VerifyType=remote时生效，默认http://localhost:8090
  "LicenseKey": "string",  // 可选：要验证的许可证密钥。留空将触发免密自动找回逻辑。
  "AppName": "string"      // 可选：主程序名称。决定本地授权文件存储在 C:\ProgramData\LicenseServer\AppName 目录下。
}
```

#### 3.3.2 响应格式（LicenseServer→主进程）

响应为JSON字符串，结构与`PipeCommunication.VerifyResponse`类一致：

```JSON

{
  "Success": true/false,   // 必选：验证是否成功
  "Msg": "string"          // 必选：验证结果描述（成功/失败原因、许可证信息等）
}
```

### 3.4 管道通信流程（主进程视角）

1. **创建匿名管道**：主进程创建两个匿名管道（一个用于写请求，一个用于读响应），并获取客户端句柄；
2. **启动子进程**：启动LicenseServer.exe，参数携带`-pipemode`及两个管道句柄；
3. **释放句柄副本**：主进程释放管道句柄的本地副本，保证子进程正常通信；
4. **发送请求**：主进程通过写管道将JSON格式的验证请求发送给子进程；
5. **接收响应**：主进程通过读管道阻塞/非阻塞读取子进程返回的JSON格式验证结果；
6. **解析结果**：反序列化响应JSON，处理验证成功/失败逻辑。

### 3.5 管道通信异常处理

- 若管道句柄无效/管道未连接，子进程会立即退出（ExitCode=2），主进程可捕获进程退出事件；
- 若请求JSON格式错误，子进程返回`Success=false`，`Msg`为“请求格式解析失败：具体原因”；
- 若验证过程中出现异常（如API调用失败、许可证不存在），子进程返回`Success=false`，`Msg`为具体异常信息；
- 管道通信超时：主进程可通过循环监听管道状态+子进程退出状态实现非阻塞等待，避免卡死。

## 4. 核心验证方法说明

- **`(bool Success, string Msg) VerifyLicense(string key)`**
  - **标准验证流程**：先获取机器码 -> 启动本地服务 -> 调用 `RemoteVerifyLicense`。
  - **用于**：GUI界面的“验证许可证密钥”按钮。
- **`(bool Success, string Msg) RemoteVerifyLicense(string key, string machineId)`**
  - **核心验证逻辑**：包含完整的许可证状态检查、设备绑定/更新流程。不负责启动服务。
  - **工作流程**：
    1. **智能免密查找**：若 `key` 为空，系统自动提取本机 `machine_id` 向服务器发起免密查询。如果当前设备曾经成功绑定过许可证，系统会自动提取绑定的许可证参与验证。
    2. **多数据清洗（最优选择）**：如果当前设备（或输入的 `key`）在数据库中关联了多条有效记录（如遇到意外的脏数据），系统会按优先级：`是否激活 (True)` -> `有效期类型 (永久)` -> `到期时间 (越晚越好)` 的顺序，智能挑选出**最优的一个许可证**进行验证，保障用户最大权益。
    3. **静默自愈**：在选择最优许可证后，对于数据库中残留的同机器码下的劣质或重复绑定记录，系统会调用 PocketBase API 静默发送 HTTP DELETE 请求予以清理。
    4. **基础校验**：校验最终选定的许可证是否存在。
    5. **状态校验**：校验许可证是否为已激活状态(`is_active`)。
    6. **有效期校验**：基于服务器当前时间，校验许可证是否在有效期内(`expires_at`)。
    7. **设备数量校验**：校验许可证已绑定设备数，是否达到最大设备限制(`max_devices`)。
    8. **本机绑定校验**：校验当前 `machine_id` 是否已绑定至该许可证(`machine_id`)。
    9. **绑定与更新**：若本机未绑定许可证，创建新的设备绑定记录；若已绑定，更新设备的关联信息（如计算机名`computer_name`）。
    10. **下发授权**：生成 AES-256 加密的本地授权文件并存入系统级 `C:\ProgramData` 对应 `AppName` 目录下，完成授权校验流程。
  - **用于**：`VerifyLicense` 方法和 `RemoteVerifyApi` 方法。

## 5. 图形用户界面 (GUI)——**`verify模式仅显示输入验证码弹窗`**

程序主窗体包含5个功能按钮：

1. **启动验证服务**：调用 `StartServer()` 方法，启动本地PocketBase服务。
2. **停止验证服务**：调用 `StopServer()` 方法，停止本地PocketBase服务。
3. **查看服务状态**：检查并弹窗显示本地服务的PID和端口。
4. **获取本机机器码**：调用 `GetMachineId()` 方法，生成并复制机器码到剪贴板。
5. **验证许可证密钥**：调用 `ShowVerifyDialog()` 方法，弹出输入框，输入密钥后执行验证（使用本地验证模式）。

## 6. 对外公开的远程验证API

**方法签名**：`public string RemoteVerifyApi(string licenseKey, string apiUrl = "")`
**功能**：供其他应用程序（如主程序）临时调用的验证接口。不依赖GUI，直接进行远程验证。
**调用方式**：实例化 `MainForm` 类后，调用此方法。

**输入参数**：

- `licenseKey` (必需): 要验证的许可证密钥字符串。
- `apiUrl` (可选): 远程API的根地址。如果传入非空值，本次调用将临时使用此地址，调用结束后恢复程序原有配置。

**返回格式** (JSON字符串)：

```json
{
  "Success": true, // 布尔值，表示验证是否成功
  "Msg": "验证通过！\n许可证key：xxxx\n机器码：xxxx\n用户：PC-NAME\n有效期：永久\n当前许可证绑定设备：2/3\n关联设备列表：PC-NAME(本机)、PC-NAME2(其它设备)", // 详细信息
  "MachineId": "e4d909c2908d" // 本次验证使用的机器码
}
```

或

```json
{
  "Success": false,
  "Msg": "API调用异常：错误信息描述"
}
```

**使用示例 (C#)**:

```csharp
MainForm licenseTool = new MainForm();
string resultJson = licenseTool.RemoteVerifyApi("your-license-key-here", "http://your-license-server.com:8090");
// 反序列化 resultJson 并处理结果
```
